Usabilidade do Software: o novo Microsoft Office

This is an archive post from 2004–2015. Some images may no longer load.

A Microsoft está a preparar a nova versão do Office. Apesar das várias versões que foram sendo desenvolvidas ao longo dos anos, com a introdução de novas funcionalidades e melhoria global das aplicações, o interface gráfico mantém-se o mesmo desde há quase 20 anos! Este ano parece que finalmente a Microsoft se começa a preocupar com esta área.

Porque é que a Microsoft decidiu finalmente alterar a forma como o utilizador interage com as aplicações? Segundo explicações dadas por Julie Larsson-Green (Gestora de Projecto do Departamento de Experiência do Utilizador da Microsoft):

Quando os computadores apareceram e as pessoas ainda estavam a aprender a usá-los, era muito fácil ter o Microsoft Word, Excel e PowerPoint todos com um interface muito semelhante. No entanto, ao longo do tempo, à medida que íamos adicionando novas funcionalidades, começou a ser mais difícil para os utilizadores encontrarem as coisas que queriam fazer com a aplicação. O Word 1.0 só tinha cerca de 100 comandos, e podiamos navegar pelo menu e vê-los a todos. Mas no Word 2003 existem mais de 1.500 comandos, muitos dos quais são difíceis de encontrar. Esta foi uma das razões principais porque nos começámos a preocupar com o interface do utilizador.

As diferenças em relação ao interface gráfico anterior são muitas e os comandos irão passar a estar agrupados seguindo uma determinada lógica. Além disso, existem botões que se destacam mais na barra de tarefas por serem maiores do que os outros. Isto permite ao utilizador ter os comandos que mais usa no topo da aplicação e de forma bem visível. No entanto, parece-me que continuam a ser botões a mais para a maioria dos utilizadores que apenas precisam de um processador de texto para escrever ou uma folha de cálculo para fazer contas...

Olá Gerhard.
A usabilidade de um website não pode ser medida por software porque é difícil o software saber o que é fácil ou difícil de usar por um humano.
Não existem interfaces iguais e por isso é dificil criar um padrão de usabilidade que se aplique a todos e tirar uma conclusão sobre se o site é usável ou não.

A usabilidade tem que ser testada usando os próprios utilizadores do website. Por vezes usa-se apenas uma checklist de usabilidade (heurística) para fazer uma avaliação por um perito, mas os testes com utilizadores reais são sempre a melhor opção.
