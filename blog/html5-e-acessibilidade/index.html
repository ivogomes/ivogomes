HTML5 e Acessibilidade

This is an archive post from 8 May 2014.

Desde a chegada do HTML5, que a web está a tornar-se um pouco mais acessível, quase sem o notarmos.

O facto de se usar simplesmente as tags <header> e <footer> para os cabeçalhos e rodapés e a tag <nav> para os menus acaba por ser uma grande ajuda para quem usa sistemas assistivos para navegar na web. Segundo os resultados de um inquérito recente da WebAIM aos utilizadores de screen-readers a maior parte dos utilizadores usa estes "atalhos" para navegar nos websites mais facilmente.

Apesar de alguns browsers ainda não terem um suporte completo às novas tags e atributos do HTML5, o que é certo é que os que já o fazem são uma grande ajuda para as pessoas que necessitam de uma ajuda extra para navegar na web.

E não nos podemos esquecer do jeitão que dá os campos dos formulários virem identificados como "email", "number", ou "date" para adaptar o teclado dos telemóveis ao tipo de informação pedida. Este é um exemplo claro de uma melhoria de usabilidade e de acessibilidade que tem um impacto universal.
