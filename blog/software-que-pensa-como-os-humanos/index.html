Software que pensa como os humanos

This is an archive post from 2004–2015. Some images may no longer load.

Esta semana reparei numa coisa engraçada quando estava a consultar o meu e-mail.
Na aplicação que eu uso, o Mail (sim, a aplicação para ver o mail chama-se mesmo Mail) havia uma mensagem que tinha muitos anexos e em vez de aparecer a quantidade exacta dos anexos aparecia a palavra "lots" que significa "muitos" ou "montes deles" em português.

Achei engraçado o facto de o sistema não funcionar apenas como uma máquina indicando o número exacto de anexos, mas sim quase como um humano em que a partir de um certo número de ficheiros, já passam a ser "montes deles". Penso que esta é uma das mais valias do software desenvolvido pela Apple: o computador não deve ser só uma máquina, deve interagir e reagir quase como se fosse uma pessoa!

Há vários outros exemplos que fazem com que o uso do computador não seja tão monótono. As mensagens são muito mais pessoais e ajudam o utilizador a tomar uma decisão. No exemplo em baixo, o texto indica-nos o que vai acontecer se não gravarmos o documento. Além disso, os botões dizem explicitamente o que fazem: "Gravar", "Não Gravar" ou "Cancelar" ao contrário de outros sistemas em que os botões apenas dizem: "Sim", "Não" ou "OK", "Cancelar"... Isto é importante porque na maior parte das vezes o que o utilizador lê primeiro são os botões porque lhe chamam mais a atenção.

Comparando estes dois avisos do sistema, qual é o mais simples de entender? E no de baixo, além dos 3 botões de tomada de decisão há ainda uma cruzinha [X] que serve para quê? Se é suposto o utilizador tomar uma decisão, penso que não deveria haver o botão [X] para fechar o aviso.

Tal como estes, há muitos outros exemplos que fazem com que a interacção de um humano com um computador seja mais "amigável". E como tal, em vez de andarmos a dar murros e pontapés nos computadores (como já vi algumas pessoas fazerem), poderiamos estar a trabalhar de forma mais eficiente e produtiva.

Permite-me corrigir-te :P. No windows basta pressionar Alt Gr + @. A tecla Alt Gr serve para escrever qualquer “3º caracter de uma tecla” (o do canto inferior direito). O símbolo de euro também é Alt Gr + E.

Quanto ao artigo que referiste sobre as designações dos botões, não sei se concordo totalmente com a utilização de textos descritivos da função. Isso prejudica a utilização frequente. Os textos dos botões (aliás como faz muito bem a Apple) devem ser sintéticos mas não abstractos (OK, Cancel).

Os botões podem ainda ser lidos por associação, o que permite reduzir a extensão das suas descrições. Por exemplo, “Guardar Alterações” e “Cancelar”. Quando lidos em conjunto, é evidente que o cancelar se refere a cancelar as alterações.

Este problema é algo que me preocupa há algum tempo. Aliás, a minha exposição no Dia Mundial da Usabilidade referiu sucintamente este tema.
