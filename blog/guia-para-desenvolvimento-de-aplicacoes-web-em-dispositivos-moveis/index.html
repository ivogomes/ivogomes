Acessibilidade

Guia para desenvolvimento de aplicações web em dispositivos móveis

This is an archive post from 2004–2015. Some images may no longer load.

Cada vez mais, o desenvolvimento de novas aplicações web tem de ter em conta as várias formas como o utilizador chega à informação. Hoje em dia estamos a assistir ao crescimento cada vez maior do acesso à informação através de dispositivos móveis com ecrãs reduzidos. Isto coloca alguns problemas à maioria dos sites que não estão preparados para serem visualizados a resoluções tão baixas. Além disso, é também importante ter em conta os tamanhos cada vez maiores dos ecrãs de computador.

Sendo assim, como desenvolver uma aplicação que possa ser vista normalmente num ecrã de 1280x960 pixels ou num telemóvel/PDA com um ecrã de 200x250 pixels?
Não é fácil, mas é possível, se tivermos folhas de estilo (CSS) diferentes para ecrã e para dispositivos móveis (que nem todos suportam). Mas depois temos também a diferença de ecrãs dos vários dispositivos. Não existe um standard. Mesmo em modelos da mesma marca é frequente encontrar vários tamanhos de ecrã diferentes.

Tamanhos de ecrãs em dispositivos móveis
Com base neste gráfico podemos ver a variedade de tamanhos disponíveis, pelo que se recomenda o uso de um tamanho máximo de 200x250 pixels.

Com base nestas dificuldades, e na sua experiência no trabalho com aplicações web e dispositivos móveis, Brian Fling desenvolveu um guia que tem como objectivo ajudar a perceber como podemos desenvolver aplicações web para este tipo de dispositivos.

Este guia está disponível gratuitamente em PDF e em breve terá uma versão em HTML.
